# Support

Blurly is an open-source project. Support happens through public project
channels when available.

## Getting Help

Before opening an issue:

1. Read [README.md](README.md).
2. Read [docs/TRADEOFFS.md](docs/TRADEOFFS.md).
3. Run `flutter analyze` and `flutter test`.
4. Check whether the issue is already known.

## Where To Open Issues

Use GitHub issues for:

- Bugs.
- Feature requests.
- Image-quality problems.
- Documentation gaps.
- Platform-specific build failures.

Issue tracker: <https://github.com/Topzee001/blurly/issues>

Do not share private images publicly. If an image is needed to reproduce a
problem, use a synthetic or public sample image when possible.

## Image Quality Reports

For blur-quality issues, include:

- Blur mode.
- Whether the subject is a person or object.
- Whether the subject is centered.
- What part of the subject/background was wrong.
- Device model and platform.
- A safe sample image or screenshot when possible.
